# PRD: 마크다운 기반 메모 & Todo 웹 앱 (Private Wiki)

## 프로젝트 개요

마크다운(.md) 파일을 기반으로 메모와 Todo 리스트를 작성·관리하는 **개인용 로컬 웹 앱**.
Next.js(React) + WYSIWYG 에디터로 개발하며, 데이터는 로컬 파일 시스템의 `.md` 파일로 저장한다.
`pnpm`을 패키지 매니저로 사용하며, `pnpm dev`로 로컬 서버를 실행한다.

---

## 확정된 요구사항

### 기술 스택

| 항목 | 결정 |
|------|------|
| 프레임워크 | Next.js (React) |
| 패키지 매니저 | pnpm |

---

### 파일 구조

```
{프로젝트 루트}/data/
├── notes/        # 메모 .md 파일 저장
└── todos/        # Todo .md 파일 저장
```

- **루트 경로**: 프로젝트 내 `data/` 폴더로 고정
- 메모와 Todo는 **별도 폴더**로 완전 분리
- UI에서도 **별도 섹션(탭 또는 사이드바 메뉴)**으로 구분
- **파일명 규칙**: `YYYY-MM-DD-{title}.md` 형식으로 자동 생성
  - 예: `2025-03-13-회의록.md`

---

### 에디터

- **WYSIWYG 방식**: 렌더링된 화면에서 바로 편집 (Notion 스타일)
- 마크다운 소스를 직접 노출하지 않고 서식이 적용된 상태로 편집
- **에디터 라이브러리**: TipTap (ProseMirror 기반, 확장성 우수)
- **자동 저장**: 입력 멈춘 후 **2초 debounce** 후 .md 파일에 저장
- **다크 모드**: 라이트/다크 테마 전환 지원

---

### Todo 기능

- Todo 체크박스 클릭 시 **실제 .md 파일을 직접 수정** (`- [ ]` → `- [x]`)
- 각 Todo 항목은 **마감일(due date)** 과 **우선순위(priority)** 메타데이터를 가짐
  - 인라인 문법으로 저장 (예: `- [ ] 할 일 @due:2025-03-20 @priority:high`)
- **마감일 UI**: 텍스트 입력 + date picker 둘 다 지원
- Todo 완료 시 파일 내용이 변경되므로 Git으로 히스토리 추적 가능

---

### 태그 기능

- 메모와 Todo 모두 `#tag` 형식의 태그 지원
- 사이드바에서 태그별 필터링 가능
- 태그는 .md 파일 본문 또는 frontmatter에 저장

---

### UI 레이아웃

```
┌──────────────────────────────────────────────────────┐
│  [메모] [Todo]          [다크모드 토글]  [새 파일 +]  │  ← 상단 네비게이션
├────────────────┬─────────────────────────────────────┤
│                │                                     │
│  파일 목록     │         WYSIWYG 에디터               │
│  ─────────     │                                     │
│  📁 notes/    │   제목                               │
│    memo1.md   │                                     │
│    memo2.md   │   □ Todo 항목                       │
│  ─────────     │   굵게  기울임  링크                  │
│  📁 todos/    │                                     │
│    todo1.md   │                                     │
│  ─────────     │                                     │
│  🏷️ 태그 목록  │                                     │
│    #work      │                                     │
│    #personal  │                                     │
│                │                                     │
└────────────────┴─────────────────────────────────────┘
```

- **사이드바** (좌): 파일 목록 + 태그 필터
- **에디터** (우): WYSIWYG 에디터 (전체 영역)
- 반응형은 MVP에서 미지원, 데스크탑 우선

## 기술 결정 메모

- **파일 시스템 접근**: Next.js API Routes에서 Node.js `fs` 모듈로 처리
- **에디터**: TipTap — WYSIWYG, ProseMirror 기반, 마크다운 직렬화 확장 사용
- **WYSIWYG → .md 변환**: `@tiptap/extension-markdown` 또는 커스텀 직렬화로 처리
- **Todo 메타데이터**: 인라인 태그 방식 (`@due:YYYY-MM-DD @priority:high`)으로 .md 파일에 저장
- **파일명**: 생성 시 `YYYY-MM-DD-{제목}.md` 자동 생성, 이후 이름 변경 가능
- **데이터 경로**: 프로젝트 루트 기준 `data/notes/`, `data/todos/` 고정
- **Git 연동**: 별도 구현 없이 사용자가 직접 Git 사용 (파일이 .md이므로 자연스럽게 가능)
